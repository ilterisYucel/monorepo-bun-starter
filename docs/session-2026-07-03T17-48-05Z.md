# Session 2026-07-03T17-48-05Z — Chart fixes & Genericness Audit

## Summary

Session focused on fixing frontend chart rendering issues caused by the WebSocket real-time data pipeline integration from the previous session, and conducting a genericness audit across all service layers.

## Fixes Applied

### `packages/ui/src/hooks/useTelemetry.ts`
- **Bug:** Map dedup key `deviceId:name:timestamp` dropped 7/8 rack entries per poll cycle because all 8 racks share the same timestamp. Also sorted ASC while rack helpers (last-wins) need DESC.
- **Fix:** Simple concat with DESC sort. Hook is now 100% generic — no deviceId, name, or rack_id knowledge.

### `apps/web/src/features/racks/hooks/useRacksData.ts`
- **Bug:** `WS_URL` missing `/ws/telemetry` path suffix → WebSocket never connected.
- **Fix:** Full URL `ws://localhost:5001/ws/telemetry`.

### `apps/web/src/features/dashboard/hooks/useDashboardData.ts`
- Same `WS_URL` bug — fixed.

### `apps/web/src/hooks/useChargeStatus.ts`
- Same `WS_URL` bug — fixed.

### `apps/web/src/hooks/useTelemetryProvider.ts`
- **Bug 1:** No WebSocket integration at all — TelemetryChart received HTTP data only (60s refresh), no live WS data stream.
- **Bug 2:** `from`/`to` time window frozen at range selection via `useMemo` — no sliding window for live monitoring.
- **Fix 1:** Added `useRealtimeTelemetry` + `useTelemetry` merge. Chart now receives sub-second WS updates.
- **Fix 2:** Replaced `useMemo` with `useRef` + `setInterval`: 1s sliding for 1m range, 5s for 1h, HTTP refetch intervals at 2s/10s/60s respectively.

### `apps/web/src/pages/SystemChartsPage.tsx`
- **Bug:** Called `useTelemetryProvider` directly without `deviceIds` → WS never connected.
- **Fix:** Switched to `useSystemTelemetry()` hook which passes `bscIds` as `deviceIds`.

### `apps/web/src/features/system-charts/hooks/useSystemTelemetry.ts`
- **Bug:** Missing `rack_id: "system"` filter → chart showed per-rack bitfield values (Battery Ready == 1) as system SOC/SOH.
- **Fix:** Restored `filters: { rack_id: "system" }`.

### `apps/web/src/pages/RacksPage.tsx`
- **Bug:** Used `useSystemTelemetry()` (system-only data) for the "Tüm Rack'ler" chart → rack_id dropdown only showed "system".
- **Fix:** Uses direct `useTelemetryProvider` with `deviceIds` from devices store, no rack_id filter. Client-side `tagFilters` dropdown now populates with all rack IDs (1-8 + system).

### `apps/web/src/features/logs/services/logsApi.ts`
- **Bug:** File never existed but was imported by `useEventAnnotations.ts` and `LogStore.ts` → web/desktop builds failed.
- **Fix:** Created `logsApi.list()` and `logsApi.create()` calling existing `/api/logs` endpoints.

### `deployment/docker-compose.yml` & `docker-compose.dev.yml`
- **Bug:** Web-service missing `REDIS_HOST` and `depends_on: redis` → ECONNREFUSED on machines without local Redis.
- **Fix:** Added env vars and dependency condition.
